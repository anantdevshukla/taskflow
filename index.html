Concepts

Event Delegation

Instead of attaching listeners to every task card individually, one listener can manage all tasks through event bubbling.

Task List Container → Single Event Listener → Detect Clicked Element → Perform Action
❌ Inefficient Approach

Attach listeners to: Task 1, Task 2, Task 3, Task 4, Task N.

Each task has its own listener in memory.

✅ Better Approach

Attach ONE listener to: #task-list

Use event.target and closest() to determine click origin.

Key Benefits:
  • Better performance: Only one listener registers in memory.
  • Less memory usage: Fewer active listeners.
  • Easier maintenance: Unified codebase handler.
  • Dynamic elements: Automatically handles future tasks added to list.
Concepts

Event Propagation

Control how events propagate through the DOM tree. Bubbling and Capturing are two ways of propagation.

Default Browser Behavior

Event Bubbling

The event starts at the target element and moves upward through its ancestors.

Visual Flow: Child Button → Parent → Grandparent Execution Order: Child, Parent, Grandparent
Capture Phase

Event Capturing

The event travels from the outermost ancestor down toward the target element.

Visual Flow: Grandparent → Parent → Child Button Execution Order: Grandparent, Parent, Child
Property Bubbling Capturing
Direction Bottom → Top Top → Bottom
Default Yes No
Listener Syntax addEventListener(type, fn) addEventListener(type, fn, true)
Bubbling is the default phase used by most applications. Event Delegation relies on Bubbling to efficiently handle events from dynamically created elements.
Concepts

Attributes vs Properties

Although they often look similar, Attributes and Properties behave differently inside the DOM.

Initial HTML State

Attribute

  • Stored in HTML markup.
  • Represents the initial value.
  • Accessed using getAttribute() / setAttribute().
HTML: <input value="Hello"> JavaScript: input.getAttribute("value") Result: "Hello"
Live DOM State

Property

  • Stored in the DOM object.
  • Represents the current live value.
  • Changes immediately on user interaction.
HTML: <input value="Hello"> User types: "Hello World" JavaScript: input.value Result: "Hello World"
HTML Attribute → Browser Parses HTML → DOM Property → User Interaction → Property Updates
Attributes describe the initial configuration of an element. Properties represent the current state of that element in memory.

01

Parsing

The browser reads raw bytes from the network, converts them to characters using the declared encoding (UTF-8), then feeds the character stream to the HTML parser.

02

Tokenization

The HTML parser scans characters and emits tokens — StartTag, EndTag, Character, DOCTYPE — using a state machine defined by the HTML5 specification.

03

DOM Tree

Tokens are consumed by the tree construction stage, assembling a live in-memory tree of Node objects: Document → Element → Text → Attribute.

04

CSSOM Tree

CSS is also parsed into tokens and assembled into a tree of style rules. Cascade, specificity, and inheritance are all resolved here before any layout occurs.

05

Render Tree

DOM + CSSOM are merged. Only visible nodes survive (display:none is excluded). Each surviving node carries its fully computed style, ready for geometry.

06

Layout & Paint

Layout (reflow) assigns exact coordinates to every box. Paint converts boxes into pixel instructions. Compositor threads draw the layers to the GPU frame buffer.
